# Launch Plan

Goal: build credibility with builders who use Claude Code, Codex, and local agents.

This is not the main revenue engine. It is a proof asset that shows serious agent workflow thinking.

## Positioning

One-liner:

> Add cross-model debate to your Claude Code agents before they finalize important work.

Short pitch:

> Agent Debate Room is a tiny open-source protocol for making one agent identity run on GPT and Claude, share memory, and stress-test major work before delivery.

## Target Audience

- Claude Code users
- Codex users
- indie hackers building agent workflows
- developers with local multi-agent setups
- AI automation consultants who want better QA

## Launch Channels

1. GitHub
2. X/Twitter
3. LinkedIn
4. Reddit: `r/ClaudeAI`, `r/ChatGPTCoding`, `r/LocalLLaMA` if relevant and non-spammy
5. Hacker News "Show HN" only after the README and example are polished

## First 48 Hours

1. Publish repo.
2. Post the X thread.
3. Post the LinkedIn story.
4. Share in one relevant community with a real explanation, not a drive-by link.
5. Ask 3 builders to try the templates and tell you where they break.
6. Add fixes quickly.

## Credibility Angle

Do not say:

- "This makes agents reliable."
- "This solves hallucination."
- "This guarantees better answers."

Say:

- "This adds structured disagreement before final answers."
- "It creates a quality gate for important agent work."
- "It made my own agents sharper because they had to argue before shipping."

## Conversion Path Back To Grove

Soft CTA:

> I am using this pattern inside Grove, where I build operational AI agent systems for ecommerce and agency workflows.

Do not hard-sell Grove in the launch post. Let credibility compound.
